Rappels sur Python et la programmation orientée objet

f-string

Une f-string est une chaîne de caractères formatée. Cela permet de créer des chaînes de caractères très simplement, sans concaténation.

Exemple :

  • version sans f-string :
a = 1
b = 2
print("a = " + str(a) + ", b = " + str(b))
  • version avec f-string :
a = 1
b = 2
print(f"a = {a}, b = {b}")

💡 Remarquez le f qui précède la chaîne de caractères : il indique qu’il s’agit d’une f-string. Les variables entre accolades sont automatiquement remplacées par leur valeur.

Paramètres optionnels dans une fonction

Dans une fonction, on peut spécifier une valeur par défaut pour des paramètres :

Exemple :

def reserverHotel(nbPersonnes = 1, petitDejeuner = False):
	# ...
	# code de la fonction
	# ...

Les paramètres qui disposent d’une valeur par défaut deviennent alors optionnels lors de l’appel à la fonction.

Exemples :

reserverHotel(3) # on spécifie explicitement le premier paramètre (nbPersonnes) ; l'autre prend sa valeur par défaut (False)
# => nbPersonnes = 3, petitDejeuner = False

reserverHotel(petitDejeuner = True) # on spécifie le second paramètre ; dans ce cas, on est obligé d'indiquer son nom (petitDejeuner) l'autre prend sa valeur par défaut (False)
# => nbPersonnes = 1, petitDejeuner = True

reserverHotel() # les deux paramètres prennent leur valeur par défaut
# => nbPersonnes = 1, petitDejeuner = False

reserverHotel(4, True) # on spécifie des valeurs pour les deux paramètres
# => nbPersonnes = 4, petitDejeuner = True

⚠️ Dans la définition d’une fonction, les paramètres optionnels doivent toujours suivre la liste des paramètres non-optionnels.

Notion de classe

Dans quasiment tous les langages de programmation, on retrouve un certain nombre de types de données (des entiers, des flottants, des chaînes de caractères…) qui servent de briques de base à la construction des programmes. Dans certains de ces langages (par exemple Java), ces “types de base” sont appelés types primitifs ; en Python, tous les types sont appelés des classes.

Une classe, c’est tout simplement un nouveau type de donnée, créé à partir d’un ou plusieurs autres types, sur lequel on définit les opérations qu’il est possible d’effectuer.

Exemple :
Supposons que l’on souhaite écrire un jeu dans lequel les personnages disposent de différentes capacités :

  • force
  • magie
  • vitesse
  • guérison

Pour gérer un ensemble de personnages, on pourrait stocker ces informations dans quatre listes séparées (une qui stocke les points de force, une autre les points de magie, etc.) :

listeForce    = [4, 5, 6, 3, 7, 1]
listeMagie    = [3, 8, 4, 1, 2, 5]
listeVitesse  = [8, 1, 7, 5, 3, 9]
listeGuerison = [1, 5, 9, 7, 5, 5]

On se rend cependant vite compte qu’il n’est pas du tout pratique de manipuler toutes ces listes, et que rien n’empêche d’insérer une valeur “aberrante”, comme -10000 dans une liste ! Il serait beaucoup plus pratique de créer un nouveau type, appelé par exemple Personnage qui regrouperait l’ensemble des capacités d’un personnage. Pour définir la nouvelle classe Personnage, nous écrirons :

class Personnage:
	# ... suite du code ...

Une classe est donc un “moule” qui permet de créer des objets. Par exemple on pourra créer un objet de type Personnage appelé gandalf, et un autre appelé frodon : on dit que gandalf et frodon sont des instances de la classe Personnage.

Créer des objets : le constructeur

En Python, toutes les classes disposent automatiquement d’une fonction (ou méthode, dans la terminologie orientée objet) nommée __init__. Il s’agit du constructeur, c’est-à-dire de la méthode qui est appelée lors de la création (ou instanciation) d’un objet. Comme son nom l’indique, elle permet d’initialiser les différents attributs de la classe ; dans notre cas, elle permet d’initialiser les capacités d’un personnage lors de sa création.

Exemple : le constructeur de la classe Personnage s’écrirait par exemple comme ceci :

class Personnage :
    def __init__(self, nom, force, magie, vitesse, guerison):
        self.name = nom
        self.strength = force
        self.magic = magie
        self.speed = vitesse
        self.recovery = guerison

💡 Nom des méthodes
Certaines méthodes, comme __init__ ont un nom qui commence et se termine par deux traits du bas (“underscore”). C’est une convention pour certaines méthodes prédéfinies en Python ; les méthodes que vous définirez dans vos classes n’ont pas besoin de suivre cette convention.

💡 Le paramètre self
Le constructeur, et toutes les autres méthodes d’une classe, doivent impérativement recevoir un premier paramètre nommé self (en réalité, on peut lui donner n’importe quel nom, mais par convention on l’appelle toujours self). C’est un moyen de désigner l’objet que l’on est en train de manipuler, et permet de distinguer une méthode (ou fonction de classe) d’une fonction “ordinaire” (pour ceux qui ont déjà fait du C++ ou du Java, c’est l’équivalent du mot-clé this).

💡 Noms des paramètres et des attributs
On peut évidemment donner les noms que l’on souhaite aux paramètres du constructeur et aux attributs de la classe ; dans l’exemple ci-dessus, les paramètres sont en français et les attributs en anglais pour insister sur le fait qu’ils ne désignent pas les mêmes variables ! Mais on pourrait tout aussi bien leur donner le même nom (c’est d’ailleurs ce que font souvent les développeurs).

Maintenant que nous disposons de la classe Personnage, nous allons pouvoir créer des objets de type Personnage, autrement dit des instances de la classe Personnage. En Python, c’est très simple, il suffit d’utiliser le nom de la classe suivi de parenthèses :

gandalf = Personnage("Gandalf", 3, 10, 5, 7)
frodon  = Personnage("Frodon", 4, 5, 4, 4)

Par ailleurs, nous pouvons déclencher une erreur si nous fournissons une valeur incorrecte dans le constructeur :

class Personnage :
    def __init__(self, nom, force, magie, vitesse, guerison):
        self.name = nom
    
        if force <= 0:
	        raise ValueError("La force ne peut pas être négative")
	    else:
			self.strength = force
    
        self.magic = magie
        self.speed = vitesse
        self.recovery = guerison

Le code suivant provoque une erreur :

boromir = Personnage("Boromir", -15, 4, 5, 6)
# ValueError: La force ne peut pas être négative

Accéder aux attributs

Pour accéder à un attribut d’un objet, c’est très simple : on utilise la notation objet.attribut ; par exemple :

gandalf = Personnage("Gandalf", 3, 10, 5, 7)
print(f"{gandalf.name} a un niveau de magie égal à {gandalf.magic}")
# Affiche : Gandalf a un niveau de magie égal à 10

Méthodes __str__ et __repr__

Par défaut, la fonction Python print ne sait pas comment afficher le contenu d’un objet, et se contente donc d’afficher l’adresse mémoire de cet objet :

>>>> gandalf = Personnage("Gandalf", 3, 10, 5, 7)
>>>> print(gandalf)
<__main__.Personnage object at 0x00000000004D0AF0>

Pas très explicite, n’est-ce pas ?

La méthode __str__ permet de spécifier comment doit être représenté un objet sous forme de chaîne de caractères ; elle est automatiquement appelée quand on utilise print.

⚠️ La méthode __str__ doit retourner une chaîne de caractères (à l’aide de l’instruction return) et non afficher une chaîne.

Supposons par exemple que nous souhaitions afficher nos personnages sous la forme :
Nom (Force : www, Magie : xxx, Vitesse : yyy, Guérison : zzz). Voici comment obtenir écrire la méthode __str__ correspondante :

class Personnage :
    def __init__(self, nom, force, magie, vitesse, guerison):
        # ... détail du constructeur, cf. ci-dessus
    
    def __str__(self):
	    return f"{self.nom} (Force : {self.strength}, Magie : {self.magic}, Vitesse : {self.speed}, Guérison : {self.recovery})"

Désormais, la fonction print sait comment afficher nos personnages :

>>>> gandalf = Personnage("Gandalf", 3, 10, 5, 7)
>>>> print(gandalf)
Gandalf (Force : 3, Magie : 10, Vitesse : 5, Guérison : 7)

Il existe une autre méthode très similaire à __str__, nommée __repr__. Il existe quelques différences entre elles, mais retenez simplement que :

  • __repr__ est utilisée par les outils de débogage de Visual Studio Code, mais pas __str__ ;
  • en l’absence de méthode __str__, la méthode __repr__ est automatiquement appelée si elle existe.

Il est donc préférable, dans le cadre de ce cours, de systématiquement écrire la méthode __repr__.
